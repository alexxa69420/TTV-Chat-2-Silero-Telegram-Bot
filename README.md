# TTV-Chat-2-Silero-Telegram-Bot

Простой бот, который читает сообщения из чата Twitch голосом через [@silero_voice_bot](t.me/silero_voice_bot) для синтеза речи.

## Что это делает

Бот подключается к чату вашего Twitch-канала, ловит сообщения от зрителей, обрабатывает их (фильтрует эмоуты, английский язык, упоминания) и отправляет в бота для синтеза речи. Полученные голосовые сообщения автоматически воспроизводятся на вашем компьютере.

## Требования

- Операционная система: Windows 10/11 \\\ Саппорт на маках и линуксах не обещается.
- Python 3.12 \\\ у меня такая как минимум.
- Доступ в интернет
- Аккаунт Twitch
- Аккаунт Telegram \\\ С подпиской на [@silero_voice_bot](t.me/silero_voice_bot)

## Настройка

### Получение данных для Telegram

1. Откройте сайт [my.telegram.org](https://my.telegram.org)
2. Войдите под своим аккаунтом Telegram
3. Перейдите в раздел "API development tools"
4. Заполните форму:
   - App title: любое название, например "TTS Bot"
   - Short name: короткое название, например "ttsbot"
   - URL: можно оставить пустым
   - Platform: выберите "Other"
5. Нажмите "Create application"
6. Скопируйте два значения: **App api_id** и **App api_hash**

### Получение токена для Twitch

1. Откройте сайт [twitchtokengenerator.com](https://twitchtokengenerator.com)
2. Выберите **Chat Bot Token** (не Custom Scope Token), чтобы токен подходил для подключения к IRC-чату.
3. В блоке **Generated Tokens** скопируйте значение **ACCESS TOKEN** (это OAuth-токен для чата).

Дальше его нужно положить **не в код и не в `config.json`**, а в файл **`.env`** в корне проекта (рядом с `TTV-Chat-2-Silero-Telegram-Bot.py`). Подробнее — в следующем подразделе.

### Файл `.env`: что куда писать

Все секреты бот читает **только из `.env`**. В `config.json` их хранить не нужно (там голоса, блеклисты и прочие несекретные настройки).

Создайте в папке проекта текстовый файл с именем **`.env`** (с точкой в начале) и заполните переменные:

| Переменная | Откуда взять | Пример / заметки |
|------------|----------------|------------------|
| `API_ID` | **App api_id** с [my.telegram.org](https://my.telegram.org) | число, без кавычек |
| `API_HASH` | **App api_hash** с того же раздела | строка |
| `TWITCH_USERNAME` | Ваш логин на Twitch (как в URL канала), **нижний регистр** | `mychannel` |
| `TWITCH_TOKEN` | **ACCESS TOKEN** с генератора | обязательно с префиксом `oauth:` — так требует IRC Twitch |
| `TWITCH_CHANNEL` | Канал, чат которого слушать | только имя канала, **без** `#` и без `twitch.tv/` |

Пример содержимого `.env` (подставьте свои значения):

```
API_ID=12345678
API_HASH=abcdef0123456789abcdef0123456789
TWITCH_USERNAME=your_login
TWITCH_TOKEN=oauth:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
TWITCH_CHANNEL=your_channel
```

Если вы скопировали ACCESS TOKEN без слова `oauth:` в начале — добавьте его вручную: `TWITCH_TOKEN=oauth:ВАШ_ТОКЕН`. При интерактивной настройке при первом запуске скрипт сам допишет префикс, если его не было.

**Важно:** не коммитьте `.env` в git и не публикуйте его — внутри полный доступ к Telegram API и к чату Twitch.

## Настройка config.json

Файл `config.json` содержит **несекретные** настройки бота (голоса, блеклисты, громкость и т.д.). Токены Telegram и Twitch задаются только в `.env` (см. раздел выше). Редактировать `config.json` можно в любом текстовом редакторе. Бот автоматически подхватит изменения без перезапуска.

### Структура файла

```json
{
  "voices": ["bandit", "arthas", "rexxar", "geralt"],
  "blacklist_users": ["streamelements", "nightbot", "jeetbot"],
  "blacklist_phrases": ["!command", "!discord", "https://"],
  "settings": {
    "target_bot": "silero_voice_bot",
    "response_timeout": 4,
    "audio_volume": 0.7,
    "default_voice": "bandit",
    "flask_host": "127.0.0.1",
    "flask_port": 8124,
    "only_highlighted": false,
    "ignore_english": true,
    "max_message_length": 350
  }
}
```

### Описание полей

**voices** — список префиксов голосов для синтеза речи. Пользователь может выбрать голос командой `!voice set <voice>`. Примеры префиксов: `bandit`, `arthas`, `anubarak`, `geralt`. Полный список доступных голосов смотрите в документации бота `silero_voice_bot`.

**blacklist_users** — список имён пользователей (в нижнем регистре), сообщения от которых будут игнорироваться (например, боты в вашем чате, или чатерсы которым вы хотите запретить озвучку).

**blacklist_phrases** — список фраз, при наличии которых сообщение будет проигнорировано (команды, ссылки). Сообщения начинающиеся на ! или / автоматически игнорируются.

**settings.only_highlighted** — если `true`, бот будет озвучивать ТОЛЬКО те сообщения, которые выделены наградой ("Highlight my message") и отправлены за баллы канала. Если `false` — читает ВЕСЬ чат.

**settings.ignore_english** — если `true`, бот автоматически удаляет английские слова и все эмоуты твича. Оставшиеся русские слова и цифры (например, от `7TV` останется только `7`) будут прочитаны.

**settings.max_message_length** — лимит символов. Если сообщение длиннее этого значения (по умолчанию 350), оно не будет озвучено, простая защита от длинных паст, увеличьте если желаете их слышать.

**settings.default_voice** — голос по умолчанию для новых пользователей и fallback-режима, если пользователь еще не выбрал голос.

**settings.audio_volume** — громкость воспроизведения (от 0.1 до 2.0).

**settings.flask_host** и **settings.flask_port** — настройки внутреннего веб-сервера. Не меняйте, если не понимаете, зачем это нужно.

## Как пользоваться

1. Запустите бот файлом `start.bat`
   - перед запуском бота `start.bat` автоматически прогоняет тесты (`pytest`);
   - если тесты не прошли, запуск бота прерывается.
2. Дождитесь сообщений в консоли:
   - `✅ Вход в Telegram` — подключение к Telegram успешно
   - `✅ Вход в Twitch` — подключение к чату успешно
   - `🔄 Worker запущен` — система обработки сообщений готова

3. Теперь все разрешенные сообщения из чата будут автоматически озвучиваться!

### Фильтрация и управление озвучкой

- Сообщения, начинающиеся с `/`, игнорируются
- Сообщения, начинающиеся с `!`, игнорируются, кроме команд `!voice`
- Пользователи из `blacklist_users` не озвучиваются
- Сообщения, содержащие фразы из `blacklist_phrases`, игнорируются
- **Удаление:** Если модератор удалит сообщение в чате Twitch до того, как бот успеет его зачитать, бот отменит озвучку этого сообщения.
- **Упоминания:** Бот автоматически вырезает любые упоминания пользователей (например, `@username`), чтобы не читать английские ники.
- **Эмоуты:** Фильтр полностью вырезает эмоуты от твича.
- **Спам скобочками:** Если зритель отправит сообщение, состоящее только из скобочек или знаков препинания (например `)))`), бот мгновенно его проигнорирует, чтобы не зависать в таймаутах.
- **Числа:** Автоматически преобразуются в слова с правильными склонениями (123 → "сто двадцать три").

### Команды выбора голоса

- `!voice list` — показать доступные голоса
- `!voice current` — показать текущий голос пользователя
- `!voice set <voice>` — выбрать голос из списка
- `!voice reset` — сбросить голос на `settings.default_voice`

## Файлы, которые создаёт бот

- `.env` — ваши секретные данные (API-ключи, токены). Никому не передавайте этот файл.

## Частые проблемы и решения

### Бот не запускается, ошибка "ModuleNotFoundError"

Не все зависимости установлены. Выполните заново внутри папки:
```
pip install -r requirements.txt
```

### Ошибка при запуске, "API_ID и API_HASH не найдены"

Файл `.env` не создан или повреждён. Удалите его и запустите бот заново — он запросит данные повторно.

### Бот подключился к Twitch, но сообщения не озвучиваются

Проверьте:
1. Правильно ли указано название канала в `.env` (без символа @)
2. Не включена ли в `config.json` настройка `"only_highlighted": true` (если включена, обычные сообщения читаться не будут)
3. Работает ли бот `silero_voice_bot` в Telegram (напишите ему вручную)

### Бот "завис" и перестал реагировать на сообщения

Нажмите `Ctrl+C` в окне консоли для корректного завершения работы бота, затем запустите его заново.
